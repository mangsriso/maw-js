# MAW.JS DATA FLOW ANALYSIS

## System Data Flow Overview

```
Claude Sessions → ~/.oracle/feed.log → FeedTailer (1s poll)
    → Engine broadcast → WebSocket → Frontend (status decay)

Tmux panes → tmux capture-pane → Engine (50ms poll)
    → Delta check → WebSocket → Frontend (terminal display)

CLI commands → ssh/tmux → Pane input
    → Capture poll → Frontend update
```

## 1. Session Management Flow
```
Tmux (local/SSH)
  → tmux.ts (listAll, capture)
  → engine.ts (broadcastSessions, 5s interval)
  → WebSocket
  → useSessions → AgentState[] → UI components
```

## 2. Message Sending Flow
```
User clicks send → useWebSocket.send({ type: "send" })
  → handlers.ts: check active Claude → ssh.sendKeys()
  → Tmux pane receives → Claude responds
  → 50ms capture poll → delta broadcast → Frontend
```

## 3. Feed Event Flow
```
Claude session logs event → ~/.oracle/feed.log
  → FeedTailer (byte-offset poll, 1s) → parseLine()
  → engine.onEvent() → WebSocket "feed" message
  → useSessions: resolveAgent → updateStatus → detectAsk
  → UI: status indicators, inbox items
```

## 4. Status Decay Logic
- Feed activity → timestamps per agent
- BUSY_TIMEOUT = 15s (last feed < 15s → busy)
- IDLE_TIMEOUT = 60s (no feed for 60s → idle)
- Special: PreToolUse without PostToolUse → stay busy

## 5. Token Tracking
```
~/.claude/projects/*/*.jsonl
  → token-index.ts: parse + sum per project/date
  → mtime-based incremental index
  → /api/tokens (summary) + /api/tokens/rate (real-time)
```

## 6. Ask Detection (Inbox)
```
Feed "Stop" (capture question text)
  + Feed "Notification" ("waiting for input/attention/approval")
  → AskItem → InboxView
  → Auto-dismiss when agent resumes (PreToolUse/SubagentStart)
```

## 7. State Persistence
- localStorage: instant hydration
- Server /api/ui-state: debounced 1s sync (cross-device)
- Asks: separate /api/asks endpoint
- Config: maw.config.json + fleet/*.json (CRUD API)

## 8. Heartbeat (15 min)
```
Collect feed events → group by parent oracle
  → Calculate token rate → Append to maw.log
  → Format: "N oracles, M actions, Z tok/min"
```

## Key Design Decisions
1. **Polling over events**: Simpler, resilient to file rotation
2. **Delta-only broadcast**: Reduces bandwidth significantly
3. **Client-side status decay**: No server state needed
4. **Worktree-aware resolution**: Feed → regex match → correct agent
5. **Safety gate**: Check for active Claude before sending
6. **Secret masking**: Bullet chars in API, restore on save
